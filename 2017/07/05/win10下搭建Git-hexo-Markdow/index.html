win10下使用Github page+Hexo+Markdow搭建个人博客

网上关于这方面的教程很多，但少有将过程中错误的原因及解决方法给出，此篇博客将对于过程中我本人遇到的问题及解决方法给出，供大家参考

安装前的准备工作

  • Github账号建立

    建立账号没人不会吧

  • Git安装（Git用来和你的github账号库进行连接）
  • Node.js安装 （因为hexo是基于Node.js环境开发的）
  • MarkdownPad(一款windows下markdown编译器)

    Github的配置

    首先打开https://github.com/，输入自己的账号密码
  • 创建代码库：

代码库的名字我们遵守既定的规则，使用”yourname.github.io”这种格式,前面的yourname输入的你的用户名

  • 安装hexo
    首先你要新建一个文件夹（这个文件夹将用来放置你博客的所有文件）然后打开它。点击鼠标右键选择”Git Bash here”然后输入

    npm install -g hexo-cli

在初始化hexo

hexo init
npm install

上面的步骤都很简单，如果你的屏幕没有输出类似失败的字眼，你就算安装成功了。

但是我们还是要验证一下自己有没有正确配置，方法很简单，输入如下代码

hexo server

也可以简写为

hexo s

等待出现如下画面：

打开你的浏览器，访问http://localhost:4040

会出现：

表示你关于hexo的基础配置已经全部成功了

但是你还没有与你的github账户进行连接呢，下面就开始进行连接吧:

先输入

git config --global user.name "yourname"
git config --global user.email "youremail"

yourname输入你的用户名

youremail输入你注册GitHub时候用的邮箱

做完这些之后我们还需要在你新建的文件夹中打开_config.yml文件，这里你可以用notepad++打开它。

打开后一直翻到最下面，看到deploy字样

修改如下:

deploy:
type: git
repo: git@github.com:yourname/yourname.github.io.git
branch: master

这里千万要注意的是每个”:”后面都要有一个空格，不然就会报错

然后再回到Git Bash,输入：

hexo g
hexo d

此时在访问你的页面

yourname.github.io

就能够看到你的页面了，如果你觉得丑，可以更换主题。

更换主题后一定要记得用

hexo clean
hexo g
hexo

先清除自己以前的配置，不然会更容易出错。
